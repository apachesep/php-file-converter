程式除錯

當你發現無論放入什麼樣到東西到 URL 裡頭，仍只有預設的頁面被載入，這表示你的伺服器可能不支援 PATH_INFO 變數(這變數是用來提供搜尋引擎友善的 URLS 位址)。 首先，開啟 application/config/config.php 這個檔案，並且搜尋 URI Protocol 的部份，他會建議你先試試其他設定值。 假如你修改後，它仍無法運作，你則需要強制要求 CodeIgniter 加上一個問號在你的 URLs 後頭。請開啟 application/config/config.php 這個檔案，接者把 Index File 底下這行:

$config['index_page'] = "index.php";

改成這樣:

$config['index_page'] = "index.php?";
